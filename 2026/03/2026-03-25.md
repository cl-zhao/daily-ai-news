### Latent Space：AI for Materials Discovery — Heather Kulik

**核心洞察**：MIT 教授 Heather Kulik 在材料科学领域深耕多年，她的研究证明 AI 能够在实验室合成之前就发现全新的化学现象。AI 筛选数千种材料的速度，比传统实验方法快上百倍。

**主要内容**：

Heather Kulik 的团队用 AI 筛选了数万种材料，发现了一种能让聚合物强度提升四倍的分子断裂机制——一个让实验化学家完全意想不到的设计。关键在于电子在特定条件下的量子力学行为，这种现象此前在聚合物材料中从未被观察到。

她指出了一个当前 AI 化学模型的明显短板：要求 AI 设计一个含 22 个原子的配体，它永远给不出正确答案，但人类化学家一秒钟就能做到。"你需要先懂化学，才能判断 AI 什么时候错了。"

在主动学习（Active Learning）领域，她的团队正在用 7 个目标同时优化金属有机框架（MOF）材料——成本、稳定性、选择性、机械强度、热稳定性——这是 AI 在多维优化问题上最有潜力的方向。

**Heather Kulik 的忠告**：
> "学化学不是因为 AI 做不了，而是因为你要能判断 AI 什么时候在胡说八道。从零开始盲目依赖 AI 是不行的。"

## 🐦 X / Twitter

### Andrej Karpathy：litellm PyPI 供应链攻击警示

前 Tesla AI 总监、OpenAI 创始团队成员 Andrej Karpathy 发出严厉警告：litellm 包（每月下载量 9700 万次）在 PyPI 上遭遇供应链投毒攻击。攻击者在上线约一小时内就成功窃取了 SSH 密钥、各大云平台（AWS/GCP/Azure）凭证、Kubernetes 配置、git 凭据、所有 API 密钥、shell 历史记录、加密货币钱包、CI/CD 密钥和数据库密码。

他指出，这次攻击幸亏因为一个 bug 导致目标机器内存耗尽崩溃，才被 MCP 插件用户发现，否则可能潜伏数天甚至数周。"供应链攻击是现代软件最可怕的威胁。每次安装任何依赖，都可能在你整个依赖树的任意深处引入恶意代码。"

**Karpathy 的结论**：这就是为什么他越来越倾向于让 LLM 直接生成简单功能，而不是引入大量第三方依赖。

### Claude (Anthropic)：Claude Code 推出 Auto Mode

Anthropic 宣布 Claude Code 全新 Auto Mode：在"每次操作都需批准"和"完全跳过权限"之间，Auto Mode 让 Claude 能够自主判断权限决策。每个工具调用前会有分类器审查安全操作，风险操作被阻止后 Claude 会换一种方式执行。

**安全机制**：分类器在每个工具调用前审查是否存在潜在破坏性操作。Anthropic 建议在隔离环境中使用此功能。Auto Mode 现已在 Team 计划上线，Enterprise 和 API 访问将在未来几天内推出。

**相关背景**：Anthropic 作为 AI 安全公司的定位，此刻发布安全增强功能，颇有意味。

### Guillermo Rauch (Vercel CEO)：SaaSpocalypse 的真实写照

Guillermo Rauch 透露 Vercel 内部几乎所有 SaaS 应用——客服、销售、营销、PM、HR、数据可视化，甚至设计和视频工作流——都已被 AI 生成的应用或 Agent 界面取代。他坦言，"我们花了多年时间试图在 Salesforce 里正确展示 Vercel 客户的状态，太多数据（数万亿消费数据点），本体论不匹配，最终 UI 非常怪异。直接生成我们需要的，比这容易得多。"

**Rauch 的判断**：Software is not going away — 但它正在成为 Agent 的 guardrails 和业务逻辑。如果你的软件不能以 Agent 能接受的方式实现 headless 连接，"you are DOA"（还没出生就死了）。
